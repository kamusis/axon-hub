# 报销字段定义参考

本文件定义了恩墨科技报销系统所有字段的数据类型、格式要求、备选值和输入提示。

Agent 在填充字段时，应参照本文件的规范执行。

---

## 字段类型说明

| 类型 | 说明 | Agent 行为 |
|------|------|-----------|
| **备选值** | 该字段只能从预定义列表中选择一个值 | AI 分析文件内容后，从列表中选择最匹配的值 |
| **自由录入** | 该字段为开放文本输入 | AI 根据文件内容和上下文生成建议文本，遵循格式要求 |

---

## 字段详细定义

### 1. 项目名称

- **CSV 列名**：项目名称
- **类型**：自由录入
- **格式**：文本
- **来源**：用户提供或从上下文推断
- **默认值**：`2026年_数据库管理服务产品群直属_ID`
- **示例**：`2026年_数据库管理服务产品群直属_ID`
- **提示**：这是报销归属的项目，通常由用户在请求时提供。如果用户未指定，默认使用 `2026年_数据库管理服务产品群直属_ID`。同一批次报销的所有记录应共享相同的项目名称。

### 2. 报销类型

- **CSV 列名**：报销类型
- **类型**：备选值
- **默认值**：`日常费用`
- **可选值**（按系统下拉列表顺序）：
  1. `销售费用`
  2. `实施费用`
  3. `售前费用`
  4. `日常费用`
  5. `办事处费用`
  6. `市场培育费用`
  7. `荣誉激励奖金`
  8. `俱乐部活动费用`
- **提示**：
  - **默认使用 `日常费用`，不要自行推断**。除非用户明确指定了其他报销类型，否则一律使用默认值

### 3. 费用类型

- **CSV 列名**：费用类型
- **类型**：备选值
- **可选值**（按系统下拉列表顺序）：
  1. `办公耗材`
  2. `一对一沟通`
  3. `市内交通费`
  4. `公交充值卡`
  5. `差旅费-补助`
  6. `差旅费-住宿`
  7. `差旅费-机票`
  8. `差旅费-汽车票`
  9. `差旅费-火车票`
  10. `差旅费-高原补助`
  11. `差旅费-其他交通费`
  12. `团队建设`
  13. `房屋及设备租金`
  14. `市场活动专项费用`
  15. `专项报销`
  16. `日常福利`
  17. `日常福利-零食`
  18. `证书费/会员费`
  19. `住宿费（非出差）`
  20. `办公家具维修费`
- **提示**：这是最关键的字段之一。根据文件内容和文件名综合判断：
  - 机票行程单 / 航空客票 → `差旅费-机票`
  - 酒店/住宿发票（出差期间）→ `差旅费-住宿`
  - 出差补助（无发票，按天数计算）→ `差旅费-补助`
  - 高原地区出差补助 → `差旅费-高原补助`
  - 打车/地铁/公交/租车/市内交通 → `差旅费-其他交通费`
  - 长途汽车票 → `差旅费-汽车票`
  - 火车票/高铁票 → `差旅费-火车票`
  - 无法归类的差旅费用 → `差旅费-其他交通费`
  - 办公耗材/打印纸/文具 → `办公耗材`
  - 一对一沟通/客户拜访相关 → `一对一沟通`
  - 市内交通（非出差）→ `市内交通费`
  - 公交卡充值 → `公交充值卡`
  - 团建活动/聚餐/团队沟通餐费 → `团队建设`
  - 房屋租金/设备租赁 → `房屋及设备租金`
  - 市场活动/展会 → `市场活动专项费用`
  - 专项报销（特殊审批）→ `专项报销`
  - 日常福利/员工福利 → `日常福利`
  - 零食/下午茶 → `日常福利-零食`
  - 证书考试费/行业协会会员费 → `证书费/会员费`
  - 非出差期间的住宿 → `住宿费（非出差）`
  - 办公家具维修 → `办公家具维修费`

### 4. 费用发生起始日期

- **CSV 列名**：费用发生起始日期
- **类型**：自由录入
- **格式**：`YYYY-MM-DD`
- **来源**：从发票/票据内容提取
- **示例**：`2026-01-17`
- **提示**：
  - 机票：取出发日期
  - 住宿：取入住日期
  - 交通：取行程开始日期
  - 补助：取出差起始日期
  - 单日费用：起始日期 = 结束日期
  - **行程合理性校验（重要）**：不要机械地使用发票开票日期。需结合行程逻辑推断实际发生日期：
    - 先根据所有交通票据（机票、火车票）建立行程时间线，确定每段行程的城市和日期范围
    - 如果某张发票的**城市**与某段行程一致，但**开票日期**不在该城市行程的日期窗口内，则该费用实际发生日期应回溯到该城市行程期间
    - 常见场景：餐费、住宿、打车等发票可能在行程结束后补开，开票日期晚于实际消费日期
    - 示例：武汉行程 5/11-5/13，武汉餐费发票开票日期 5/19 → 实际发生日期应为 5/12 或 5/13（武汉行程期间）

### 5. 费用发生结束日期

- **CSV 列名**：费用发生结束日期
- **类型**：自由录入
- **格式**：`YYYY-MM-DD`
- **来源**：从发票/票据内容提取
- **示例**：`2026-01-25`
- **提示**：
  - 单日费用（机票、单次交通等）：结束日期 = 起始日期
  - 住宿：取退房日期
  - 补助：取出差结束日期
  - 多日费用：取最后一天日期
  - **行程合理性校验**：同字段4（费用发生起始日期），需根据行程时间线推断实际结束日期，而非机械使用发票开票日期

### 6. 报销金额

- **CSV 列名**：报销金额
- **类型**：自由录入
- **格式**：纯数字，保留两位小数，不含货币符号
- **来源**：从发票/票据内容提取
- **示例**：`2987.00`
- **提示**：
  - 从发票上找到"金额"、"价税合计"、"总价"等字段
  - 注意区分"金额"和"税额"，报销金额通常为价税合计
  - 机票注意是否含保险费、燃油附加费
  - 补助类费用：天数 × 每日补助标准（见下方出差补助规则）

### 7. 单据数

- **CSV 列名**：单据数
- **类型**：自由录入
- **格式**：整数
- **来源**：自动填充
- **默认值**：`1`
- **提示**：
  - 每张图片/PDF 通常对应 1 张单据。如果一张图片中包含多张发票，如实填写数量。
  - **补充单据不计入单据数**：行程报销单、明细清单、订单截图等与核心发票配套的补充文件，不单独计数。只计核心票据（如电子发票本身）。
  - 示例：滴滴电子发票 + 滴滴行程报销单 = 2个文件，但只按 1 张单据计。

### 8. 费用说明

- **CSV 列名**：费用说明
- **类型**：自由录入
- **格式**：文本，简洁明了
- **来源**：AI 根据文件名 + 文件内容生成
- **示例**：
  - `南京-北京往返机票`
  - `9天出差补助`
  - `8天无酒店住宿补助`
  - `南京市某某酒店住宿3晚`
  - `南京站至禄口机场打车`
- **提示**：
  - 航空类：出发地-目的地 + 机票
  - 住宿类：城市 + 酒店/住宿
  - 交通类：起点-终点 + 交通方式
  - 补助类：差旅补助和住宿补助各一条记录，费用说明简洁写明天数和类型即可，不交叉提及
  - 尽量包含城市、具体服务、数量等关键信息

### 9. 出差申请

- **CSV 列名**：出差申请
- **类型**：自由录入
- **格式**：文本（出差申请编号）
- **来源**：用户提供
- **示例**：`202602245263`
- **提示**：**必须由用户提供，不可默认填充。** 如果用户未提供出差申请编号，必须暂停处理并询问用户获取。同一批次报销的所有记录应共享相同的出差申请号。

### 10. 出差天数

- **CSV 列名**：出差天数
- **类型**：自由录入
- **格式**：整数
- **来源**：从交通票据推算
- **示例**：`11`
- **提示**：
  - **计算规则**：出差天数 = 第一个交通票据日期 → 最后一个交通票据日期（含首尾）
  - 只用交通类票据（机票、火车票、汽车票）计算，不用住宿、餐费等
  - 交通票据的首尾天然界定了出差的起止——第一张是出发日，最后一张是返程日
  - 例如：去程高铁 4月5日，回程高铁 4月15日 → 出差天数 = 15 - 5 + 1 = 11天
  - 住宿发票可辅助验证（住宿晚数 ≤ 出差天数）
  - **特殊情况**：如果目录中包含与本次出差无关的交通票据（如上一次出差的回程票），需结合行程方向和目录名判断，排除不属于本次出差的部分
  - 如果票据不完整无法推算，应向用户确认
  - **仅补助记录填写此字段，其他类型的报销记录（机票、火车票、住宿、餐费等）均留空**

### 11. 销售活动编号

- **CSV 列名**：销售活动编号
- **类型**：自由录入
- **格式**：文本
- **来源**：用户提供的上下文
- **默认值**：（留空）
- **提示**：除非用户明确提供了销售活动编号，否则留空。

### 12. 活动类型

- **CSV 列名**：活动类型
- **类型**：备选值
- **可选值**：

### 13. 报工单

- **类型**：自由录入

- **来源**：用户提供的上下文

## 出差补助计算规则

出差补助是**无发票、按天数计算**的费用，需生成独立记录，费用类型为 `差旅费-补助`。

### 补助类型

#### 1. 差旅补助（含交通补助 + 误餐补助）

- **中国大陆地区**：100 元/人/天（误餐补助 50 元 + 交通补助 50 元）
- **中国港澳台地区**：100 元/人/天
- **计算方式**：按实际出差天数（含往返当天），以票面出发与到达时间为准；当天来回计 1 天
- **注意**：内部培训不享受此补助

#### 2. 住宿补助（不住酒店的补助）

- **标准**：100 元/天
- **条件**：员工出差期间选择住在亲友家，不报销酒店费用
- **可与差旅补助同时享有**（住宿补助是替代住宿费报销的补偿）

### 记录生成规则

- **差旅补助**与**住宿补助**应分为**两条独立记录**，费用类型均为 `差旅费-补助`
- **差旅补助**：天数 = 出差天数 × 100 元，日期范围 = 出差起止日期
- **住宿补助**：晚数 = 出差天数 - 1 - 有酒店住宿的晚数，日期范围 = 出差起止日期。其中"出差天数 - 1"是总住宿晚数（返程日当天返回，不需住宿）
- **单据数**：补助类记录无实体票据，填 `0`
- **费用说明示例**：
  - 差旅补助：`11天出差补助`
  - 住宿补助：`9天无酒店住宿补助`（11天出差 - 1晚返程日 - 1晚有酒店 = 9晚）
- **注意**：既然差旅补助和住宿补助是分开的两条记录，费用说明中不应交叉提及另一项补助。差旅补助只写出差天数，住宿补助只写无酒店天数
- **判断哪些晚有酒店**：从住宿费发票的入住日期推断，不在住宿发票覆盖范围内的晚数即为无酒店晚数。注意：返程日当天返回不需住宿，总住宿晚数 = 出差天数 - 1

### 向用户确认

- 首次处理报销时，如果用户未主动告知有出差补助需求，应询问是否需要添加补助记录
- 如果用户告知有补助需求，但无法确定哪些天住了酒店，应主动询问或从住宿发票推断

## CSV 输出规范

- **编码**：UTF-8
- **分隔符**：逗号 `,`
- **换行符**：`\n`（通用格式）
- **引号**：字段含逗号或换行时用双引号包裹
- **第一行**：表头（字段名）
- **空字段**：留空（两个逗号之间无内容）
